# LOCA · 내 지도 탭 시각 리디자인 구현 브리프

## 배경

LOCA 내 지도 탭(`MapsListScreen.jsx`)의 **시각 디자인만** 정돈하는 작업입니다. 정보 구조 · 우선순위 · 컴포넌트 구성 · 인터랙션은 **건드리지 않습니다**. 옷을 다림질하는 수준의 마감 작업입니다.

## 작업 스코프

**범위 안**
- 헤더 압축 (LOCA 로고 + 부제 제거, 액션 아이콘화)
- 탭 스위처 segmented 스타일
- 지도 카드의 region 컬러 시스템 적용
- 장소 탭 필터 칩 3단 그룹화
- 장소 row 컴팩트화 + 이모지 노출
- 빈 이름("새 장소") 안티패턴 처리

**범위 밖**
- 정보 구조 변경 (지도/장소 두 탭 그대로)
- 검색 위치 변경
- 데이터 흐름 변경
- 라우팅 변경
- 새 섹션 추가
- 컴포넌트 시그니처 변경

## 화면별 헤더 차이 (중요)

**두 화면의 우측 액션 아이콘 구성이 다릅니다.**

| 액션 | A 지도 탭 | B 장소 탭 |
|---|---|---|
| 지도 가져오기 (다운로드 아이콘) | ✓ ghost | ✓ ghost |
| **새 지도 만들기 (+ 아이콘)** | ✓ **primary 오렌지** | ✗ 노출 안 함 |
| 알림 (벨 아이콘) | ✓ ghost + dot | ✓ ghost + dot |

> **B 화면에서 "새 지도 만들기"를 노출하면 안 됩니다.** 장소 목록 보고 있는 맥락에서 새 지도 생성은 흐름이 끊김. 지도 목록 볼 때만 자연스럽게 등장하는 액션.

> **FAB와의 분리 유지**: 헤더의 + 버튼은 새 지도 컨테이너 생성, 하단 FAB의 + 버튼은 현재 지도에 마커 추가. 모양이 같아도 위치(상/하단)가 달라 사용자가 헷갈리지 않음. 절대 합치지 마세요.

DESIGN.md §14 Don'ts: **"새 장소"를 기본 이름으로 방치 금지**.

```jsx
const displayName = place.name?.trim() || '이름 없는 장소 · 직접 입력하기';
const isEmpty = !place.name?.trim();

<div className={`place-name ${isEmpty ? 'empty' : ''}`}>
  {displayName}
</div>
```

`empty` 클래스가 적용된 row를 탭하면 **편집 시트가 이름 필드에 포커스된 상태로 열려야** 합니다 (사용자 입력 유도).
