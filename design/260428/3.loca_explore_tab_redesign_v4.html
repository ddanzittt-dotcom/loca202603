설계 노트 · 탐색 탭 리디자인

  • 섹션 2개로 단순화 — 기존 4섹션(EXPLORE 히어로 + PICKED + TOGETHER + NEARBY) → 2섹션(TOGETHER 모두의 지도 + NEARBY 근처 행사). PICKED 둘러보기 좋은 지도 제거. 탐색 탭의 핵심을 "우리 동네"로 좁힘 — 추천 지도 큐레이션은 홈 탭의 PICKED와 역할 중복이었음.
  • EXPLORE 히어로 다크 카드 해체 — 다크 그린 큰 박스 안에 검색바가 갇혀있던 구조 → 인사 텍스트(투명) + 큰 검색바(흰 카드)로 분리. 이제 검색이 시각적으로 가장 강한 진입점. 검색바 우측에 ⌘K 키 힌트로 검색 어포던스 강화.
  • 인기 검색어 칩 신설 — 검색을 즉시 시도하지 않는 사용자를 위한 진입 보조. 가로 스크롤 칩(벚꽃 / 성수동 / 한옥 / 감성카페 / 제주 동쪽 / 독립서점)에 첫 번째만 ↑ 트렌딩 표시. 검색 박스 → 칩 → 모두의 지도 → 근처 행사로 자연스러운 시선 이동.
  • "모두의 지도" 용어로 통일 — "함께 남긴 지도" → 모두의 지도. LOCA에 이미 정착된 용어를 사용하여 다른 화면(헤더, 메뉴 등)과 일관성 유지. 부제는 "동네 사람들이 함께 만들어가는 지도"로 정체성 강화.
  • 모두의 지도 카드 강화 — 핵심 진입점이므로 시각 비중 키움. mint bg + 우상단 blob + 실제 미니맵 시각화(거리 + 강 + 마커 8개) + 실제 동네 이름("안서동")과 통계("지난 주 326곳 추가")로 살아있는 느낌.
  • 단일 CTA로 액션 단순화 — 기존 "모두의 지도 열기 / 지도 떠나가기"는 모호하고 충돌. 변경: 전폭 "모두의 지도 둘러보기" primary 버튼 한 개. "내 장소 추가" 버튼 제거 — 모두의 지도에 진입한 다음 자연스럽게 가능한 액션이라 진입 단계에 두 액션을 노출하면 사용자가 망설임. 첫 번째 결정은 "갈까 말까" 한 가지여야 함.
  • "근처에서 기록해볼 만한 행사" — "근처에서 열리는 지도"는 모호. 사용자 입장에서는 "이게 지도냐 행사냐"라는 카테고리 혼란이 있음. 변경: 근처에서 기록해볼 만한 행사로 명확히 — 행사임을 분명히 하고, "기록해볼 만한"이라는 동사로 사용자 행동(LOCA의 핵심 가치)을 유도.
  • 이벤트 카드 위계 정돈 — 우측 상태 뱃지를 3종류로 구분: D-10(secondary 그린, 임박), 진행중(mint, 라이브 닷 애니메이션 가능), 곧 시작(warm). 이벤트 카드 내부에 기간 + 제목 + 위치/스탬프 메타 3단 위계. 썸네일에 이모지 노출하여 시각 다양성.
  • 위치 chip + 정렬 상태 — 행사 섹션 위에 위치 칩(천안 동남구) + "총 28개 · 거리순" 한 줄 추가. 사용자가 어떤 범위로 보고 있는지 즉시 인지하고 칩 탭으로 위치 변경 가능.
  • "더보기 (28개)" 톤 강화 — 기존 약한 텍스트 → 점선 보더 카드 형태로 어포던스 강화. 숫자만 회색, 텍스트는 description 톤으로 시각 위계 분리.
  • 탭 정체성 명확화 — H1을 "탐색"으로 (홈/내 지도/탐색/프로필 패턴 통일). EXPLORE 태그는 보조로만 작게.
  • 변경 안 한 것 — 4번째 탭이 active인 탭바 위치, FAB 위치, 알림 벨, 핵심 정보 우선순위는 그대로. 옷만 다림질한 수준이라 데이터 모델 변경 불필요.

설계 노트 · 검색 활성 화면 (B, C)

  • 진입 트리거 — A 화면의 검색바를 탭하면 풀스크린 검색 모드로 전환. 헤더(탐색 + 알림)가 사라지고 그 자리에 검색 input + 취소 버튼이 옴. 일반적인 iOS/Android 검색 패턴.
  • 활성 input 시각 차이 — 평소 input은 옅은 보더(rgba(0,0,0,0.06)), 활성 시 secondary 그린 보더 + 0 0 0 3px rgba(45,74,62,0.08) 글로우 ring. 사용자가 "지금 입력 가능 상태"임을 한눈에 인지. 캐럿 컬러는 primary 오렌지로 LOCA 정체성. placeholder는 "지도, 에디터 검색"으로 검색 대상을 명시.
  • 입력 전 상태 (B) — 빈 input일 때 두 블록: 최근 검색(개인 이력, 시계 아이콘 + 텍스트 + × 삭제 + 모두 지우기) + 인기 검색(글로벌 트렌드 8위 + 등락 변화 표시). "최근"이 위에 있는 이유는 개인 이력이 더 클릭 가능성 높음.
  • 인기 검색 등락 표시 — ▲▼— 변화 + NEW 뱃지로 트렌드 살아있게. 1-3위는 primary 오렌지 숫자로 강조, 4위부터는 회색. 명조체 숫자로 LOCA 톤 유지.
  • 입력 중 상태 (C) — 입력값이 1자 이상이면 즉시 2개 카테고리로 결과 노출: 지도(mint, 메인) + 에디터(amber, 보조). 장소 카테고리는 제외 — 사용자가 탐색 탭에서 검색하는 의도는 큐레이션된 지도를 발견하거나 관심 에디터를 팔로우하는 행위. 개별 장소는 결국 지도의 일부이므로 따로 노출하면 결과가 분산됨. 카테고리당 최대 4개 미리보기 + "더보기 →"로 확장.
  • 검색어 하이라이팅 — 결과 텍스트의 일치 부분을 <mark> 태그로 감싸 warm bg + warm fg로 강조. 특히 에디터의 핸들에서 부분 매치(seongsu_lover, sungsoo_walks)도 잡아서 검색 정확도 어필.
  • 장소 카테고리 제외 이유 — LOCA의 장소는 지도 안에 속하는 마커이지 독립 콘텐츠가 아님. 사용자가 "성수"를 검색하면 "성수 카페 산책"이라는 큐레이션 지도가 더 가치 있는 결과. 개별 장소(예: "대림창고 성수점")가 필요하면 그 지도에 들어가서 보면 됨. 결과를 카테고리별로 분산시키는 것보다 의미 있는 단위(지도)로 모아주는 것이 탐색 경험에 유리.
  • "클리어 버튼"과 "취소 버튼" 분리 — input 안의 × 버튼은 입력값만 지우기(input은 활성 유지), 우측 "취소" 텍스트 버튼은 검색 모드 자체 종료. iOS 표준 패턴.
  • 탭바 유지 — 검색 모드에서도 탭바는 그대로. 사용자가 검색을 중단하고 다른 탭으로 바로 이동 가능. 모달이 아닌 같은 탭의 다른 화면이라는 신호.
  • 구현 팁 — A→B 전환 시 검색바 위치를 position: absolute로 두고 layout transition. autofocus + 최근 검색 데이터 lazy load. 입력 중에는 debounce 200ms로 results API 호출.
