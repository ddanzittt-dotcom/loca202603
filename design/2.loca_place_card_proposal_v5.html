피커 동작 사양

  • — 장소 카드의 이모지 버튼 탭 시 열림 (경로/범위에는 이모지 없음)
  • — 검색 + 형태별 탭 + 단일 6열 그리드
  • — 활성 탭: warm bg + 하단 2px primary 오렌지 마커
  • — 선택: primary 1.5px 보더 + warm bg 이중 강조
  • — 저장: emoji string("☕") 형태로 DB에 한 글자만
  • — 렌더: native 시스템 이모지 폰트 (iOS/Android 각 OS 느낌 자동)

설계 노트 · v4 최종

  • 3가지 마커 타입 전체 커버 — 이전까지 장소(핀)만 다뤘던 편집 화면을 장소 · 경로 · 범위 3종 전부로 확장. 각 타입의 성격에 맞는 필드 구성 (장소=이모지, 경로·범위=선 종류).
  • 입력 라벨 전면 적용 — 이모지·색상·선 종류 등 모든 영역에 명확한 라벨 + 힌트 문구. 예: "색상 · 지도에 표시될 핀 색". 사용자가 각 영역이 무엇을 조작하는지 즉시 이해.
  • 이모지 버튼 구성 변경 — 단순 44px 버튼 → 44px 버튼 + 현재 이모지 이름(커피/치킨 등) + 안내 문구 3단 구조. 탭 유도성과 현재 상태 가독성 둘 다 확보.
  • 경로·범위는 이모지 없음 — 선형/면적 도형은 아이콘으로 요약하기 어렵고, 시각 언어가 색상+선 종류로 충분. 대신 측정 정보(2.1km, 약 12,400㎡)를 mint 카드로 readonly 표시.
  • 선 종류 선택 UI — 3개 버튼(실선 / 짧은 점선 / 도트) 가로 배치, 각 버튼에 SVG 선 미리보기 + 텍스트. 활성 버튼은 warm tint. iOS Pills 스타일을 LOCA 톤으로 로컬라이즈.
  • 색상 팔레트 — 6가지 (오렌지 · 다크그린 · 블루 · 레드 · 브라운 · 퍼플) + "직접" 커스텀. 각 카드의 기본 선택값이 다름 (장소=오렌지, 경로=다크그린, 범위=브라운).
  • '한 줄 소개' + 메모 역할 분리 — 상단의 한 줄 소개는 장소 요약(리스트·팝업 카드에서 미리보기로 쓰일 짧은 문장). 긴 감상·방문 기록·시간대별 경험은 자연스럽게 메모로 유도. 두 영역의 입력 대상이 시각적으로 구분되어 "어디에 뭘 써야 할지" 판단 비용이 0에 수렴.
  • 메모 첨부 플로우 — 하단 📝 탭 시 메모 입력창 표출 → 저장 시 작성 시간이 자동 기록되어 메모 리스트에 추가. 팝업 카드(v8)의 memo-preview 패턴과 동일한 좌측 secondary 2px 액센트 보더.
  • rows 축소로 역할 힌트 — 한 줄 소개의 rows="2"로 짧게 유지, placeholder도 "예: 창가 자리 좋은 카페" 같은 짧은 예시로 설정. 사용자가 자연스럽게 긴 내용은 메모 쪽으로 이동.
  • 퀵 툴바 3종 — 📷 🎤 📝 36×36 ghost 버튼. 첨부 개수는 primary 오렌지 카운터 뱃지로 표시. 툴바 자체는 얇아 공간 차지 최소.
  • 비작성자 열람 모드 제외 — 팝업 카드(이전 v8)에서 이미 담당하는 영역이라 장소 카드 레이어에서 중복 제거. 역할 분리: 팝업 카드=요약 열람+가져오기, 장소 카드=편집 전용.
  • 이전 레이어와의 일관성 — 팝업 카드(v8)에서 정립된 토큰(마커 컬러 · 버튼 위계 · 14px 카드 반경 · 10px 버튼 반경 · font-weight 500 상한)을 전부 승계. 두 레이어가 연속된 디자인 언어.
