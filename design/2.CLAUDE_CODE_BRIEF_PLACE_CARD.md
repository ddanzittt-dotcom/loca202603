# LOCA · 장소 · 경로 · 범위 카드 구현 브리프

## 배경

LOCA는 위치 기반 커뮤니티 지도 앱입니다. 이번 작업은 **마커(장소 · 경로 · 범위)를 추가 · 편집하는 바텀시트 카드**를 구현합니다.

> **이전 작업과의 관계**
> - **팝업 카드** (이미 구현됨): 지도에서 마커를 탭했을 때 뜨는 요약 미리보기. 가져오기 · 수정 · 길찾기 등 진입점 역할.
> - **장소 / 경로 / 범위 카드** (이번 작업): 팝업에서 "수정" 또는 "새로 만들기"로 진입하는 **편집 전용 시트**.
>
> 두 레이어가 한 세트의 디자인 언어로 동작합니다.

## 작업 스코프

마커 추가 · 편집 바텀시트 컴포넌트만 다룹니다. **비작성자 열람 모드는 이 레이어에 없습니다** — 팝업 카드에서 이미 담당합니다. 이번 스코프는 오직 **작성자 편집**.

## 참고 파일

작업 시작 전 이 세 파일을 모두 열어보세요.

1. `design/DESIGN.md` — 디자인 시스템 토큰 (컬러 · 타이포 · 반경 · 컴포넌트 규칙)
2. `design/loca_place_card_proposal_v5.html` — **시각적 정답지** (모든 케이스 구현된 HTML 시안)
3. `design/loca_popup_proposal_v8.html` — 이전 레이어(팝업 카드) 시안 · 디자인 언어 통일을 위해 참조

시안은 구현의 기준입니다. 픽셀 값, 간격, 컬러, 아이콘, 문구까지 v5를 따르되, DESIGN.md와 충돌하면 DESIGN.md를 우선합니다.

## 선행 단계 (권장 순서)

1. 리포지토리 구조 탐색 (언어 · 프레임워크 · 스타일 시스템 확인)

## 공통 디자인 사양

### 컬러 토큰 (DESIGN.md 참조)

기본 토큰은 팝업 카드와 동일. 추가된 **핀 색상 팔레트 6종 + custom**:

```
--pin-orange:  #FF6B35   (기본 · 장소)
--pin-green:   #2D4A3E   (기본 · 경로)
--pin-blue:    #3B82F6
--pin-red:     #EF4444
--pin-brown:   #854F0B   (기본 · 범위)
--pin-purple:  #A855F7
+ "직접" 커스텀 color picker
```

### 반경

- 시트 (바텀시트): `18px 18px 0 0` (상단만 라운드)
- 시안에서는 전체 18px로 렌더링되지만, 실제 구현은 상단만
- 카드 내부 요소(입력·버튼·칩): 10px
- 태그 칩: 8px
- 색상 칩 / 아바타: 50% (원형)

### 헤더

- 상단 4px handle bar (드래그 영역)
- 제목(Nanum Myeongjo 15px) + meta 라인 10px (예: `작성자 · 박경주` / `천안지도 · 편집 중`)
- 우측 28×28 닫기 버튼 (ghost)

### 필드 라벨 패턴 (중요)

**모든 입력 영역 상단에 라벨 + 힌트 문구** 필수. 사용자가 각 영역이 무엇을 조작하는지 즉시 이해하도록.

```
라벨 형식: "주라벨" + <span class="hint">· 부가 설명</span>
예시:
  - "이모지 · 장소를 대표하는 아이콘"
  - "색상 · 지도에 표시될 핀 색"
  - "선 종류 · 경로의 선 스타일"
  - "태그 · 엔터로 추가"
  - "한 줄 소개 · 어떤 장소인지 짧게"
```

### 하단 액션 버튼

```
[삭제(danger ghost · 100px 고정)]  [저장(primary filled · flex:1)]
```

빈 상태(새로 만드는 중)에서는 `[삭제]` 대신 `[닫기(secondary ghost)]`.

## 장소 (핀) 카드 상세

### 필드 구성 (위에서 아래로)

1. **이모지 선택**
   - 44×44 원형 버튼 + 현재 이모지 (font-size 24px, native emoji font 스택)
   - 버튼 우하단에 "⋯" 인디케이터 (피커 열림 신호)
   - 우측에 현재 이모지 이름(예: `커피`, `치킨`) + 안내 문구 (`탭해서 다른 이모지로 변경`)
2. **색상 선택** (28×28 원형 칩 6개 + `직접` 커스텀 chip)
3. **이름** (bold 15px input)
4. **설명** (모두의 지도) 또는 **한 줄 소개** (내 지도)
   - 모두의 지도: textarea `rows=3`, placeholder "이 장소에 대해 알려주세요"
   - 내 지도: textarea `rows=2`, placeholder "예: 창가 자리 좋은 카페"
5. **(내 지도) 태그** — inline chip input
6. **(내 지도) 미디어 블록** — 사진 / 음성 / 메모 (첨부가 있을 때만 렌더)
7. **(내 지도) 퀵 첨부 툴바** — `📷 🎤 📝` 36×36 ghost 버튼 + 카운터 뱃지
8. **액션 버튼**

### 한 줄 소개 vs 메모 역할 분리 (중요)

**내 지도의 장소·경로·범위 카드 모두 동일 원칙**:

| 영역 | 역할 | 입력 방식 |
|---|---|---|
| 한 줄 소개 | 장소 요약 (리스트 · 팝업 프리뷰에서 쓰일 짧은 문장) | 상단 textarea `rows=2` |
| 메모 | 방문별 시점 기록 (감상 · 팁 · 경험 등) | 하단 `📝` 버튼 → 시트 → 타임스탬프 자동 |

긴 글 작성이 예상되면 사용자는 자연스럽게 메모 쪽으로 이동하도록, **한 줄 소개는 `rows=2`로 짧게 유지**해야 합니다.

## 경로 카드 상세

### 필드 구성

1. **색상 선택** (장소와 동일한 팔레트)
2. **선 종류 선택** — 3개 버튼 가로 배치
   - `실선` / `짧은 점선` / `도트`
   - 각 버튼 내부: SVG 선 미리보기 + 텍스트
   - 활성 버튼: `warm bg + warm fg`, 비활성: `#FAF5EE bg + 0.5px border + 설명 컬러 텍스트`
   - SVG stroke-dasharray 값:
     - 실선: 없음
     - 짧은 점선: `4 3`
     - 도트: `0.1 4` (stroke-linecap round)
3. **이름**
4. **설명 / 한 줄 소개** (지도에 따라 달라짐)
5. **(내 지도) 태그**
6. **측정 정보 카드** (readonly)
   - mint bg, 10px padding
   - 경로 아이콘 + `전체 길이 <strong>2.1 km</strong> · 도보 약 22분`
   - 경로 길이는 서버 계산 or 네이버 지도 SDK에서 가져오기
7. **(내 지도) 미디어 블록 + 퀵 툴바**
8. **액션 버튼**

### 이모지 없음

경로는 선형 도형이라 이모지로 요약하기 어렵고, 색상 + 선 종류만으로 충분한 시각 언어를 가집니다. **경로 카드에는 이모지 버튼을 렌더하지 마세요**.

## 범위 카드 상세

경로와 구조 거의 동일. 차이점만:

- **기본 색상**: `--pin-brown` (#854F0B)
- **측정 정보**: 길이 대신 **면적** (`범위 면적 약 12,400㎡ · 도보 6분 권역`)
- 아이콘: 점선 사각형 (`stroke-dasharray: 3 2`)
- **이모지 없음** (경로와 동일)

선 종류는 범위의 **외곽선 스타일**로 작용 (범위 내부는 같은 색의 반투명 fill).

## 모두의 지도 vs 내 지도 차이

| 항목 | 모두의 지도 | 내 지도 |
|---|---|---|
| 설명 필드 | `설명` textarea rows=3 | `한 줄 소개` textarea rows=2 |
| 태그 | 없음 | 있음 (chip input) |
| 미디어 블록 | 없음 | 있음 (사진 · 음성 · 메모) |
| 퀵 첨부 툴바 | 없음 | 있음 (`📷 🎤 📝`) |
| 시트 메타 | `작성자 · 박경주` | `천안지도` 또는 `천안지도 · 편집 중` |

모두의 지도 카드는 **훨씬 단순**합니다. 공용 매핑이라 설명까지만 편집하고, 기록(사진 · 음성 · 메모)은 없습니다.

## 이모지 피커 바텀시트 (장소 카드 전용)

### 열림 조건

장소 카드의 **이모지 버튼을 탭했을 때**. 경로 · 범위에서는 열리지 않음 (이모지 필드 자체가 없음).

### 구조

```
[Handle bar]
[제목: "이모지 선택"]                    [×]
[검색 인풋: "이모지 검색 (예: 꽃, 커피)"]
[형태별 탭 7개: 🕘 😊 🍰 🌿 🐰 ✨ 🏠]
[6열 그리드 (세로 스크롤)]
```

### 형태별 탭 (절대 용도별 아님)

인스타그램 이모지 키보드 레퍼런스:

```
🕘 최근 사용
😊 표정·기분
🍰 음식·디저트
🌿 식물·자연
🐰 동물·생명
✨ 심볼·무드
🏠 오브젝트·공간
```

**"카페" · "음식점" 같은 용도별 카테고리는 절대 만들지 마세요.** 사용자가 "이 장소는 카페니까 커피잔을 써야 한다" 같은 강요된 매핑에 갇히게 됩니다.

### 이모지 그리드

- 셀 `aspect-ratio: 1`, radius 12px, 배경 `#FAF5EE`
- 이모지 font-size 22px, native emoji 폰트 스택 사용
- 선택: primary 1.5px 보더 + warm bg 이중 강조
- 호버: `scale(1.06)` 마이크로 인터랙션
- **최소 60+개 이모지 제공**, 특히 20–30대 사용자 감성 (💕 🫶 🫖 🫧 🥐 🧋 🌸 🌷 🦋 🐰 🎀 💌 🕯️ 등)

### 저장 포맷

- DB에 **이모지 string**(`"☕"`) 하나만 저장. 별도 아이콘 ID 테이블 만들지 마세요.
- 렌더는 시스템 native emoji font 스택 (`'Apple Color Emoji', 'Segoe UI Emoji', 'Noto Color Emoji', sans-serif`)
- iOS/Android 각 OS의 native 느낌이 자동으로 획득됨

## 메모 첨부 플로우 (내 지도 전용)

### 엔트리 포인트

1. 하단 퀵 툴바의 `📝` 버튼 탭
2. 메모 블록이 없으면 바로 시트 열림, 있으면 블록 + 기존 메모 유지

### 메모 입력 시트

- 별도 바텀시트 (또는 간이 모달)
- 구성: handle bar + 제목 "메모 추가" + textarea (여러 줄) + [취소] [저장] 버튼
- 저장 시 **작성 시간 자동 기록** (예: `04.18 · 금요일 저녁`)

### 메모 엔트리 (리스트 아이템)

```
┌─────────────────────────────────────────┐ 
│ 04.18 · 금요일 저녁                 [×] │  ← left 2px secondary accent border
│ 아빠랑 처음 와봤는데 진짜 맛있었음.       │
└─────────────────────────────────────────┘
```

- 좌측 2px secondary 액센트 보더 (팝업 카드의 memo-preview와 시각 언어 통일)
- 우상단 `×` 버튼으로 개별 삭제
- 가장 최근 메모가 상단에 표시

## 선 종류 버튼 사양 상세

경로 · 범위 카드에서 공통 사용:

```html
<button class="line-btn active">       <!-- 또는 그냥 .line-btn -->
  <span class="line-preview">
    <svg viewBox="0 0 28 10" fill="none" stroke="currentColor" 
         stroke-width="2" stroke-linecap="round">
      <line x1="2" y1="5" x2="26" y2="5"/>     <!-- 실선: dasharray 없음 -->
      <!-- 짧은 점선: stroke-dasharray="4 3" -->
      <!-- 도트: stroke-dasharray="0.1 4" stroke-width="2.5" -->
    </svg>
  </span>
  실선
</button>
```

3개 버튼은 가로로 `flex: 1`씩 균등 분할, gap 6px.

## 색상 팔레트 UI 사양

```
[●][●][●][●][●][●][직접 🟧]
 orange  green  blue  red  brown  purple  custom
```

- 각 칩: 28×28 원형, 0.5px border `rgba(0,0,0,0.06)`
- 선택된 칩: outer ring (`position: absolute; inset: -4px; border: 1.5px solid primary`)
- "직접" 칩:
  - 백색 배경, 0.5px dashed border secondary 톤
  - 내부: 라벨 "직접" + 14×14 swatch (현재 선택된 커스텀 색상 미리보기)
  - 탭 시 **네이티브 color picker** 또는 **HSL color wheel 모달** 열기
  - 반환된 색상을 swatch에 반영

## 인터랙션 사양

| 요소 | 동작 |
|---|---|
| 이모지 버튼 탭 | 이모지 피커 시트 열림 |
| 피커에서 이모지 선택 | 시트 닫힘 + 장소 카드에 반영 + "최근 사용"에 추가 |
| 색상 칩 탭 | 선택 상태 토글 (단일 선택) |
| "직접" 칩 탭 | color picker 모달 열림 |
| 선 종류 버튼 탭 | 단일 선택 토글 |
| 태그 입력 + 엔터 | chip 추가 (색상은 warm/mint/amber 중 순차 or 랜덤) |
| 태그 chip × 버튼 | 해당 chip 제거 |
| `📷` 버튼 | OS 갤러리/카메라 접근 → 선택 후 사진 썸네일 추가 |
| `🎤` 버튼 | 녹음 시작 시트 (녹음 · 정지 · 재생 · 저장) |
| `📝` 버튼 | 메모 입력 시트 → 저장 시 타임스탬프 + 리스트 추가 |
| 썸네일 × 버튼 | 해당 미디어 제거 (확인 모달 선택적) |
| 측정 정보 카드 | readonly, 탭 시 동작 없음 (또는 상세 설명 tooltip) |
| 저장 버튼 | 전체 필드 유효성 검증 후 서버 전송 |
| 삭제 버튼 | 확인 모달 → 삭제 API 호출 |

## 접근성

- 아이콘 전용 버튼(`×`, 퀵 툴바)은 `aria-label` 필수
- 색상 칩은 색상만으로 구분되지 않도록 `aria-label`(예: "오렌지 선택됨")
- 선 종류 버튼에는 현재 선택 상태를 `aria-pressed`로 표시
- 이모지는 screen reader가 읽을 수 있도록 그대로 텍스트로 둠
- Tab 순서: 헤더 → 이모지 → 색상 → 선 종류 → 이름 → 한 줄 소개 → 태그 → 미디어 → 툴바 → 액션

## 완료 기준 (Acceptance Criteria)

구현 완료 전 스스로 체크:

- [ ] 모두의 지도: 장소 · 경로 · 범위 3개 카드 전부 동작
- [ ] 내 지도: 장소 · 경로 · 범위 3개 카드 전부 동작
- [ ] 내 지도 장소 빈 상태에서 첨부 상태로 자연스럽게 전환
- [ ] 모든 입력 영역에 **라벨 + 힌트 문구** 표시
- [ ] 이모지 피커: 형태별 탭 7개 · 60+개 이모지 · 검색 동작
- [ ] 경로/범위 카드에는 **이모지 버튼 미노출**
- [ ] 선 종류 3개 옵션 동작 (실선 / 짧은 점선 / 도트)
- [ ] 경로 측정 정보(km · 도보 분) 자동 계산 표시
- [ ] 범위 측정 정보(㎡ · 권역) 자동 계산 표시
- [ ] 내 지도에 메모 첨부 플로우 완성 (`📝` → 시트 → 저장 → 리스트)
- [ ] 한 줄 소개 `rows=2`로 짧게 유지 (긴 글은 메모로 유도되는 UX)
- [ ] 바텀시트 상단만 `border-radius: 18px 18px 0 0`
- [ ] 모바일 뷰포트(360~390px)에서 레이아웃 깨짐 없음
- [ ] DESIGN.md 토큰 외 하드코드 컬러 없음
- [ ] 팝업 카드(v8)와 시각 언어 일치 (마커 컬러 · 버튼 위계 · 반경 · font-weight 500 상한)
- [ ] HTML 시안(v5)과 픽셀 단위 비교 검증

## 진행 방식 제안

큰 작업이므로 나눠서 진행:

1. **1단계** — 리포 탐색 + 기존 팝업 카드 · 디자인 토큰 파악
2. **2단계** — 공통 컴포넌트 (Sheet · FieldLabel · ColorChips · LineStylePicker · EmojiButton · TagInput · MeasureInfo · MediaBlock · MemoEntry · AttachToolbar)
3. **3단계** — 이모지 피커 바텀시트 (검색 + 형태별 탭 + 그리드)
4. **4단계** — 장소 카드 (모두의 지도 + 내 지도 · 빈/첨부 상태 분기)
5. **5단계** — 경로 카드 (모두의 지도 + 내 지도)
6. **6단계** — 범위 카드 (모두의 지도 + 내 지도)
7. **7단계** — 메모 첨부 플로우 (입력 시트 + 저장 + 리스트 반영)
8. **8단계** — 인터랙션 + 상태 관리 연결 (가져오기 · 저장 · 삭제 API)
9. **9단계** — QA + Acceptance Criteria 체크

각 단계마다 중간 커밋해주세요.

## 결정·질문 필요 항목 (임의 판단 금지)

다음은 서비스 정책 · 기술 스택 관련이라 정답이 없습니다. **임의로 진행하지 말고 질문으로 남겨주세요**:

1. **경로 길이 · 범위 면적 계산** — 네이버 지도 SDK 내장 함수? 자체 계산? 서버 계산?

9. **모두의 지도 "설명" 필드 길이 제한** — 공용 매핑이니 너무 길면 피드 공간 낭비
