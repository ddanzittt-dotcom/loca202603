# LOCA · 지도 팝업 카드 구현 브리프

## 배경

LOCA는 위치 기반 커뮤니티 지도 앱입니다. 두 가지 지도 모드가 있고, 각 지도에서 사용자가 마커를 탭하면 팝업 카드가 열립니다.

- **모두의 지도** — 공용 매핑 (누구나 기여 가능)
- **내 지도** — 개인 매핑 (작성자의 개인 기록, 구독자가 볼 수 있음)

## 작업 스코프

이 작업은 **지도 팝업 카드 컴포넌트**만 다룹니다. 마커 자체, 지도 렌더링, 편집 화면(상세보기)은 이번 스코프가 아닙니다.

## 참고 파일

작업 시작 전 반드시 이 두 파일을 먼저 읽어주세요.

1. `design/DESIGN.md` — 디자인 시스템 토큰 (컬러, 타이포, 반경, 컴포넌트 규칙)
2. `design/loca_popup_proposal_v8.html` — 시각적 레퍼런스 (모든 케이스가 구현된 HTML 시안)

HTML 시안은 구현의 정답지입니다. 픽셀 값, 간격, 컬러, 아이콘까지 이 파일을 기준으로 삼고, DESIGN.md와 충돌하면 DESIGN.md를 우선하세요.

## 선행 단계 (진행 순서 권장)

1. 리포지토리 구조 탐색 (언어·프레임워크·스타일 시스템·기존 컴포넌트 규칙 파악)
2. 디자인 토큰 파일 존재 여부 확인 (Tailwind config · CSS 변수 · 디자인 시스템 패키지 등)
3. 기존 지도/팝업 관련 코드 위치 파악
4. 그 다음 구현 착수

기존 컨벤션을 최대한 따라주세요. 프로젝트에 없는 라이브러리를 임의로 추가하지 마세요.

## 전체 구성 요약

### 지도 × 권한 × 마커 타입 매트릭스 (총 12 케이스)

| 지도 | 권한 | 마커 타입 |
|---|---|---|
| 모두의 지도 | 작성자 / 비작성자 | 핀 · 경로 · 구역 |
| 내 지도 | 작성자 / 비작성자 | 핀 · 경로 · 구역 |

HTML 시안에서는 이 중 8개 케이스를 대표로 보여주지만, 구현 시 12개 전부 대응되어야 합니다.

## 공통 디자인 사양

### 컬러 (DESIGN.md 참조, 요약)

| 용도 | 값 |
|---|---|
| primary | `#FF6B35` |
| secondary | `#2D4A3E` |
| background | `#FAF5EE` |
| card | `#FFFFFF` |
| text-primary / secondary / tertiary / desc | `#1A1A1A` / `#888` / `#aaa` / `#666` |

### 마커 타입별 컬러 페어

| 타입 | 배경 | 전경 |
|---|---|---|
| 핀 | `#FFF4EB` (warm-bg) | `#993C1D` (warm-fg) |
| 경로 | `#E1F5EE` (mint-bg) | `#085041` (mint-fg) |
| 구역 | `#FAEEDA` (amber-bg) | `#633806` (amber-fg) |

### 반경

- 카드: `14px`
- 버튼 / 이미지 썸네일 / 미디어 칩: `10px`
- 타입 태그 / 일반 태그: `8px`

### 폰트

- Pretendard 기반, weight 상한 `500` (600 이상 사용 금지)
- 제목 헤더의 섹션 제목은 명조 계열 (프로젝트에 이미 있는 서체 규칙 따르기)

### 헤더 공통 규칙

- `align-items: center`로 수직 중앙 정렬 (상단 치우침 금지)
- 좌측: 34×34 타입 아이콘 박스 (radius 10px, 배경은 타입 컬러 bg)
- 중앙: 제목 (핀·구역은 제목만, 경로는 태그 row에 길이 표기)
- 우측: 액션 버튼들 + 닫기 버튼

## 모두의 지도 (공용 매핑) 사양

### 레이아웃 순서

```
[Header: 타입 아이콘 + 제목 + 액션들 + 닫기]
[Tag row (경로 타입만: 길이 km)]
[Description 본문]
[Photo strip (가로 스크롤, 하단 날짜)]
[Memo section (별도 영역, 배경 톤다운)]
```

### 헤더 액션 버튼

**작성자:**
- `[가져오기]` (warm tint, 북마크+ 아이콘 + 텍스트)
- `[수정]` (icon-only 28×28 ghost, 펜 아이콘, `title="수정"`)
- `[닫기]` (기본 ghost)

**비작성자:**
- `[가져오기]` 또는 `[저장됨]` (토글 상태)
  - 가져오기: warm tint, outline 북마크+ 아이콘
  - 저장됨: mint tint, filled 북마크 아이콘
- `[수정요청]` (icon-only 28×28 ghost, 깃발 아이콘, `title="수정요청"`)
- `[닫기]`

### 사진 스트립

- 각 아이템: `<media-item>` = 56×56 썸네일 + 하단 9px 날짜 라벨 (`MM.DD`)
- **작성자만** 마지막에 점선 테두리 '추가' 칩 노출 (56×56, secondary 톤)
- **비작성자는 추가 칩 없음** (사진 기여는 메모 입력 카메라로만)
- 썸네일이 많을 때 마지막 썸네일 우하단에 `+N` 카운터

### 메모 섹션

- 상단 구분선 + 배경 `rgba(250,245,238,0.4)`로 영역 분리
- 헤더: `메모` 라벨 + 개수
- 입력 필드: 좌측 카메라 아이콘(ghost) + 텍스트 input + `추가` 버튼(primary orange)
- 입력 중 사진 첨부 시: input 아래 36×36 썸네일 가로 배치 + 각 썸네일 우상단 × 제거 뱃지
- 메모 리스트 아이템 구조:
  - 24×24 아바타
  - 작성자명 + (내 메모일 때 mint 뱃지) + 작성 시간
  - 텍스트 본문
  - 첨부 사진이 있으면 텍스트 아래 48×48 썸네일 row (4장 이상이면 마지막에 `+N` 오버레이)

## 내 지도 (개인 매핑) 사양

### 레이아웃 순서

```
[Header: 타입 아이콘 + 제목 + 액션들 + 닫기]
[Tag row (사용자 태그들, 경로는 맨 앞에 길이 km 추가)]
[Content 본문]
[Media stack: 사진 / 음성 / 메모 프리뷰]
```

### 헤더 액션 버튼

**작성자:**
- `[상세보기]` (secondary filled 버튼, `>` chevron)
- `[닫기]`

**비작성자 (구독자 뷰):**
- `[닫기]`만

### 미디어 스택 (세로 배치, 각 섹션 간격 8px)

**사진 row (가로 스크롤):**
- 각 아이템: 76×76 썸네일 + 하단 9px 날짜
- 초과 시 마지막 썸네일 `rgba(26,26,26,0.45)` 오버레이 + `+N`
- 스크롤바 숨김 (`scrollbar-width: none`)

**음성 row (가로 스크롤):**
- 각 카드: **100×48 고정폭** (한 화면에 3개 딱 맞게)
  - 좌측 28×28 재생 버튼 (secondary 배경, 흰색 삼각형)
  - 우측 상단: 파형 8칸 (`mint-fg` 컬러, 재생된 칸은 opacity 1, 나머지 0.75)
  - 우측 하단: 9px 타임스탬프 (`0:45` 형식, opacity 0.75)
- 카드 하단: 날짜 라벨 (사진과 동일한 스타일)

**메모 프리뷰:**
- 배경 `#FAF5EE`, 좌측 secondary 2px 액센트 보더
- 텍스트 2줄 clamp (`-webkit-line-clamp: 2`)
- 하단: 작성일 + `더보기 →` (secondary 컬러, 우측 정렬)
- 우상단 ghost 메모 아이콘

### 태그 row

- 사용자 태그 + 경로의 경우 **맨 왼쪽에 길이 칩** (`1.4km`, mint 컬러, tabular-nums)
- 태그 칩: 8px radius, 9px 폰트, weight 500

---

## 타입 아이콘 (34×34 박스 안)

- **핀**: 지도 핀 SVG, fill `#FF6B35`, 내부 원 `#FFF4EB`
- **경로**: 지그재그 라인 SVG, stroke `#0F6E56`, width 1.8
- **구역**: 점선 라운드 사각형 SVG, stroke `#854F0B`, `stroke-dasharray: 3 2`

## 버튼 컬러 위계 (중요)

사용자 인지 부담을 줄이기 위한 위계를 반드시 유지:

| 계층 | 예시 | 스타일 |
|---|---|---|
| 최고 참여 | 메모 `추가` | primary orange filled |
| 주요 액션 | `가져오기` | warm tint filled |
| 완료 상태 | `저장됨` | mint tint filled |
| 보조 액션 | 수정 / 수정요청 | secondary ghost (icon-only) |
| 기본 | 닫기 | neutral ghost |

## 인터랙션 사양

| 요소 | 동작 |
|---|---|
| 가져오기 탭 | 서버에 복제 API 호출 → 성공 시 `저장됨` 상태로 전환 |
| 저장됨 탭 | 확인 모달 → 가져오기 취소 (내 지도에서 해당 엔트리 제거) |
| 수정 / 수정요청 탭 | 편집 화면 또는 요청 모달로 이동 |
| 상세보기 탭 | 내 지도 엔트리 상세 화면으로 이동 |
| 닫기 탭 | 팝업만 닫음 (지도 상태 유지) |
| 메모 `추가` | 텍스트 + 첨부 사진을 서버 전송 → 리스트 상단 추가 |
| 메모 카메라 아이콘 탭 | 갤러리/카메라 접근 → 선택 시 input 아래 썸네일로 추가 |
| 썸네일 탭 | 풀스크린 갤러리 뷰어 열기 |
| 음성 재생 버튼 탭 | 인라인 재생 (한 번에 하나만 재생, 다른 음성 재생 시 이전 것 정지) |
| 사진 / 음성 가로 스크롤 | 터치 스와이프 지원, 스크롤바 숨김 |

---

## 접근성

- 아이콘 전용 버튼에는 `title` 속성 + `aria-label` 필수
- 색상 대비 AA 이상 (기존 토큰은 이미 검증됨)
- 음성 플레이어 재생/정지 상태를 aria-pressed로 표시
- 키보드 Tab 순서: 헤더 액션들 → 본문 → 메모 입력 → 메모 리스트

## 완료 기준 (Acceptance Criteria)

구현 완료 전 스스로 체크:

- [ ] 모두의 지도 6케이스 (핀·경로·구역 × 작성자·비작성자) 모두 동작
- [ ] 내 지도 6케이스 모두 동작
- [ ] 모든 사진·음성 아이템에 하단 날짜 표기
- [ ] 핀·구역은 제목 아래 타입 태그 없음 (좌측 아이콘으로 충분)
- [ ] 경로만 길이 km가 태그 row 맨 왼쪽
- [ ] 모두의 지도 비작성자는 상단 '사진 추가' 칩 미노출
- [ ] 내 지도 비작성자는 헤더에 `닫기`만
- [ ] 음성 카드 한 화면에 3개 노출 + 4번째부터 스와이프
- [ ] 가져오기 ↔ 저장됨 토글 동작
- [ ] 메모 입력 시 사진 첨부 → 서버 전송 흐름 완성
- [ ] 모바일 뷰포트 (360~390px) 레이아웃 깨짐 없음
- [ ] DESIGN.md 토큰 외 임의 하드코드 컬러 없음
- [ ] HTML 시안과 시각적 비교 검증 (간격·크기·컬러)

## 진행 방식 제안

큰 작업이므로 나눠서 진행해주세요:

1. **1단계** — 리포 탐색 + 공통 컴포넌트 (PopupCard 껍데기, TypeIcon, ActionButton, DateLabel)
2. **2단계** — 모두의 지도 구현 (헤더 3종 액션 + 사진 스트립 + 메모 섹션)
3. **3단계** — 내 지도 구현 (미디어 스택: PhotoRow, VoiceRow, MemoPreview)
4. **4단계** — 권한 분기 로직 연결 + 상태 관리
5. **5단계** — 인터랙션 (가져오기 토글, 음성 재생, 사진 첨부 플로우)
6. **6단계** — QA 및 완료 기준 체크

각 단계가 끝날 때마다 중간 커밋해주세요. 커밋 메시지는 프로젝트 컨벤션을 따르고, 어떤 단계인지 명시해주세요.
