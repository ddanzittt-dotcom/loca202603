04

모션 · 카피 · 아이코노그래피

"임시 느낌"의 핵심 원인을 지우는 마지막 마감 요소.

모션 원칙 · 4가지

① 반응성 (150ms)
버튼 hover · 칩 토글 · 입력 포커스. 즉각적이지만 매끈하게.
② 진입/퇴장 (240ms ease-out)
바텀시트 · 페이지 전환 · 토스트. 끝에서 부드럽게 멈춤.
③ 캐릭터 모션 (spring)
핀 등장 시 살짝 바운스. EmptyState 캐릭터는 천천히 떠다님.
④ 자제 (절대 금지)
화면 전체가 회전·확대·페이드인. 모션은 양념, 본문 아님.

카피 보이스

❌ 임시 느낌 (제거 대상)
· "준비 중이에요"
· "TBD" / "Coming soon"
· "데이터를 불러오는 중입니다."
· "오류가 발생했습니다."
✅ LOCA 톤 (대체 카피)
· "곧 만나요 🌱" + 알림 신청 버튼
· 빈 화면엔 캐릭터 + "첫 지도를 만들어볼까요?"
· 로딩엔 스켈레톤 + (지연 시) "조금만 기다려주세요 ☕"
· 에러엔 "잠깐, 인터넷이 깜빡였어요. 다시 시도?" + 재시도 버튼

1

npm i lucide-react · 기능 아이콘 일괄 교체 → 체감 변화 ★★★★★

3
컬러·라운드·그림자 토큰 통일
CSS 변수로 통일, inline style 제거 → 체감 변화 ★★★

EmptyState · 로딩 · 에러 → 캐릭터 적용
화면 단위로 점진 마이그레이션 → 체감 변화 ★★★★
