## 1. 한눈에 보는 결론

- LOCA의 B2B/B2G 수익모델 가설은 현재 기준으로 타당한가?: **보통**
- 지금 바로 팔 수 있는가?: **조건부 가능** (운영형 파일럿/프로젝트형)
- 가장 큰 병목 1개: **캠페인-예산-정산-성과증빙으로 이어지는 “상품 레이어” 부재**
- 가장 먼저 고쳐야 할 것 1개: **검증 가능한 성과 패키지 정의(체크인 신뢰 강화 + KPI/리포트 표준화)**
- 90일 내 가장 현실적으로 팔 수 있는 상품 1개: **“행사 참여증빙 운영 패키지(Managed Pilot)”**

## 2. 현재 제품의 실제 가치 흐름 요약

### 현재 LOCA가 유저에게 제공하는 핵심 가치

- 행사/로컬 지도를 통해 참여자가 QR 진입 후 체크인/완주/설문까지 수행 가능  
  근거: `loca202603/src/screens/SharedMapViewer.jsx:399`, `loca202603/src/hooks/useEventMapData.js:246`
- 공유 후 저장/재사용 흐름(share -> save -> import)이 제품 내에서 연결됨  
  근거: `loca202603/src/screens/SharedMapViewer.jsx:388`, `loca202603/supabase/migrations/015_map_saves.sql:7`, `loca202603/supabase/migrations/016_map_lineage.sql:6`
- 운영자는 별도 대시보드에서 참여 퍼널/댓글/공지/지도 운영 가능  
  근거: `loca-dashboard/src/App.jsx:27`, `loca-dashboard/src/lib/dashboardService.stats.js:252`

### 현재 LOCA가 만들어내는 행동 데이터/참여 증거

- 행동 로그: `view_logs` 이벤트(`map_view`, `checkin`, `completion`, `share_click`, `map_save` 등)  
  근거: `loca202603/src/lib/analytics.js:15`, `loca202603/src/lib/analytics.js:142`
- 참여 증거: `event_checkins`, `event_completions`, `survey_responses`, `event_comments`  
  근거: `loca202603/supabase/migrations/011_daily_cap.sql:235`, `loca202603/supabase/migrations/011_daily_cap.sql:293`, `loca202603/supabase/migrations/012_event_comments.sql:42`
- 운영 조직 스코프: `organizations`, `organization_members`, `can_view_map`, `can_manage_map`  
  근거: `loca202603/supabase/migrations/022_dashboard_tenant_rbac.sql:9`, `loca202603/supabase/migrations/022_dashboard_tenant_rbac.sql:333`, `loca202603/supabase/migrations/022_dashboard_tenant_rbac.sql:358`

### 이것이 B2B/B2G 가치로 연결되는 지점

- 참여 퍼널/완주율/공유-저장 전환 지표를 리포트로 제시 가능  
  근거: `loca-dashboard/src/lib/dashboardConfig.js:55`, `loca-dashboard/src/lib/dashboardService.stats.js:217`
- 지도 운영 안정성(발행 리비전/롤백) 제공  
  근거: `loca202603/supabase/migrations/017_publication_revisions.sql:7`, `loca-dashboard/src/App.jsx:459`

### 연결이 끊기는 지점

- 파트너 예산 집행 단위(캠페인/보상/정산/청구)가 없다.
  - 코드 검색 결과 `campaign/coupon/settlement/billing/invoice/payout/budget`가 main/dashboard 모두 0건.
- 체크인 서버 검증이 약해 “검증 가능한 성과” 신뢰도가 부족하다.
  - 거리 판정은 클라이언트 버튼 제어 중심이며, 서버 함수는 좌표 저장 중심.
  - 근거: `loca202603/src/screens/SharedMapViewer.jsx:406`, `loca202603/supabase/migrations/011_daily_cap.sql:233`
- 리포트가 PDF 중심이라 구매자 실무(정책 보고/정산 자료/추가 분석)에 필요한 구조화 export가 약하다.
  - 근거: `loca-dashboard/src/lib/reportGenerator.js:229`, `loca-dashboard/src/lib/reportGenerator.js:312`, `loca-dashboard/src/pages/DashboardPage.jsx:763`

### 현재 구조 평가

- **“앱 + 운영 대시보드가 있는 프로토타입과 상품 사이 단계”**
- 사용자 기능은 강하지만, 구매자가 돈을 내는 핵심 체인(예산 -> 집행 -> 증빙 -> 정산 -> 재집행)이 미완성.

## 6. 실제로 고려 가능한 BM 옵션 최소 3개

### BM 1) 행사 참여증빙 운영 패키지 (Managed Campaign Ops)

- 타깃 고객: 행사 대행사, 지역 상권 운영팀, 관광 사업자
- 구매 이유: 오프라인 행사 성과를 “참여 퍼널 데이터”로 증빙해야 함
- 제공하는 결과물: 지도 구축, 체크인 운영, 참여/완주/설문 리포트, 운영 가이드

### 왜 다른 BM보다 우선순위가 높은가

- SaaS 구독형(BM2)이나 성과형(BM3)은 정산/보상/감사 레이어가 더 필요해 초기 딜 클로징 시간이 길다.
- BM1은 “지금 있는 기능 + 운영 역량”으로 90일 내 레퍼런스 확보 가능성이 가장 높다.

### 최소 기능 조합

- 행사 지도 발행/QR 공유
- 체크인/완주/설문 데이터 수집
- 공지/댓글 모더레이션
- 참여 퍼널 리포트(PDF + CSV)
- 체크인 서버 검증 1차 하드닝(반경/정확도)

### 어떤 고객에게 먼저 가야 하는가

- 의사결정이 빠른 **행사 대행사/관광 프로그램 운영사**(중간 규모, 단기 집행 경험 다수 보유)

### 어떤 증빙 자료가 있어야 닫을 수 있는가

- 파일럿 1~2건의 전/후 비교 지표
- 운영 중 장애/부정참여 대응 로그
- 결과 리포트 샘플(PDF + 원본 CSV)

### 이 상품의 1문장 세일즈 피치

- “LOCA는 행사 참여를 지도에서 실시간으로 수집하고, 체크인-완주-설문 성과를 바로 보고서로 납품하는 운영형 솔루션입니다.”

### 3문장 제안서 요약

- LOCA는 QR 진입부터 체크인/완주/설문까지 참여 흐름을 한 지도에서 운영합니다.  
- 행사 종료 후에는 참여 퍼널과 만족도 데이터를 리포트로 제공해 성과 설명 책임을 줄여줍니다.

## 8. B2B 파일럿 제안안

- 가장 적합한 B2B 파일럿 시나리오 1개: **상권/쇼핑몰 주말 스탬프 투어**
- 대상 고객 예시: 지역 복합쇼핑몰 마케팅팀, 상권연합회
- 파일럿 목적: 방문 유도 + 점포 체류/동선 데이터 확보
- 제공 범위:
  - 행사 지도 1개(체크포인트 8~15개)
  - QR 랜딩/공지/댓글 운영
  - 체크인/완주/설문 수집
  - 운영 리포트 납품
- 사용자 경험 흐름:
  - QR 스캔 -> 지도 열람 -> 점포 체크인 -> 완주 -> 설문 -> 종료 안내
- 운영 플로우:
  - 킥오프(목표/KPI 합의) -> 맵 세팅 -> 현장 QA -> 운영 모니터링 -> 종료 리포트
- 측정 KPI:
  - 유입 세션, 체크인 참여자, 완주율, 평균 체류시간, 만족도, 공유-저장 전환율
- 산출 리포트:
  - 운영 PDF 1부 + 원본 CSV(개인 식별자 마스킹)
- 가격 가설:
  - 4~6주 파일럿 총 600~1,500만원(구축+운영 포함)
- 본계약 전환 조건:
  - 핵심 KPI 2개 이상 목표 달성
  - 운영 장애/어뷰징 이슈 허용 범위 내
  - 다음 캠페인 요구사항이 제품 로드맵과 정합

## 9. B2G 파일럿 제안안

- 가장 적합한 B2G 파일럿 시나리오 1개: **지자체 관광 스탬프형 걷기 프로그램**
- 대상 기관 예시: 관광재단, 문화재단, 지자체 관광과
- 파일럿 목적: 분산 방문 유도와 시민 참여 성과 증빙
- 제공 범위:
  - 코스형 행사 지도 2~3개
  - 공지/댓글 정책 운영
  - 참여 데이터 수집 및 기관 보고 포맷 리포트
- 시민/참여자 경험 흐름:
  - 공공 채널 QR/링크 진입 -> 체크포인트 방문 체크인 -> 완주 -> 만족도 설문
- 운영 플로우:
  - 기관 목표 합의 -> 지도/체크포인트 확정 -> 현장 운영 -> 주간 브리핑 -> 결과 보고
- 측정 KPI:
  - 참여자 수, 코스별 완주율, 재방문율, 시간대 분포, 설문 만족도
- 산출 리포트:
  - 기관 제출형 PDF + CSV + 운영 이슈 로그 요약
- 가격 가설:
  - 6~8주 파일럿 총 1,000~3,000만원(구축+운영+보고 포함)
- 본사업 전환 조건:
  - 기관 목표 대비 참여/완주 지표 달성
  - 데이터 신뢰성(부정참여 비율 관리) 확보
  - 차기 분기/연간 반복 프로그램 예산 연결 가능성 확인
