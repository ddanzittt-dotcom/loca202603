# LOCA 데이터 대시보드 전체 구성안 (/admin)

## 배경

LOCA에 쌓이는 데이터(장소 카드·기록·동네 태깅·협업·소비 로그)는 **지역 데이터 자산**이다.
커뮤니티(모두의 지도)는 현재 앱에서 빠져 있으므로, 자산은 **개인 채집 기록의 익명 집계**에서 나온다
(Strava 모델 — 개인 기록 도구가 집계 데이터 자산을 만든다). 협업(함께 만들기, map_collaborators)은 유지.

## 대시보드의 3가지 역할

1. **계기판** — 서비스·데이터 자산이 건강하게 자라는지 매일 확인 (운영자용)
2. **자산 증명** — "이런 데이터가 쌓였다"를 숫자·지도로 보여주는 자료 (투자·제휴 미팅용)
3. **상품화 소재** — 동네 리포트·트렌드 콘텐츠의 원천 (B2G/마케팅용)

## 자산 우선순위 (커뮤니티 없는 전제)

1. **NEW FIND (미등록 장소)** — 카카오 로컬에 없는 곳을 사용자가 먼저 기록.
   CollectSheet가 `"새발견"` 태그로 이미 저장 중 → 집계만 하면 됨 (057 반영).
2. **재방문 기록 시계열** — 리뷰(1회성)와 달리 기록 도구라 같은 장소 반복 기록 발생 → 단골/생명주기 신호.
3. **동네×카테고리 채집 밀도·추이** — 054(region 태깅) 기반. 뜨는 동네 선행지표.
4. **실보행 코스(route)** — 사람들이 좋아서 걷는 길.
5. 유통 데이터(공유/QR) — 커뮤니티 부재로 볼륨 작음, 후순위.

## 전체 섹션 구성 (7개 영역)

### ① 운영 현황 — 서비스 규모 ✅ 구현 (056 get_admin_overview)
- 가입자(+7일/30일), 지도(발행·7일), 장소·기록(타입 분해), 메모·팔로우, 조회수(7일), 순 방문자(30일 세션)

### ② 지역 자산 — 핵심 경쟁력 ✅ 구현 (057 get_admin_insights)
- NEW FIND 총수·7일 / 동네 랭킹 TOP15(카드·7일·새발견) / 동네 태깅률 / 인기 태그 TOP15

### ③ 성장 추이 ✅ 기본형 (8주 바차트+표) / 🔲 확장: 기간 선택, 일별, 동네별 필터, 누적 곡선

### ④ 협업 (함께 만들기) ✅ 구현
- 협업 지도·참여자 / 초대 수락률(수락·거절·대기) / 협업으로 만든 카드(주인 아닌 참여자 등록분)

### ⑤ 품질·건전성 ✅ 구현 / 🔲 확장: 리텐션 코호트, 유저 세그먼트(헤비/일반/휴면), 기록 길이·음성 수
- 7일 활성 채집자 / 유저당 평균 카드 / 반복 기록 카드 / 사진 있는 카드·기록

### ⑥ 유통·소비 ✅ 기본형 / 🔲 확장: 인기 지도 TOP10, 공유→가입 전환
- 유입 소스 분포(링크/카카오/QR/직접) / 지도 저장·좋아요

### ⑦ 공간 시각화 🔲 전체 예정 (Phase 2) — 미팅용 "한 장"
- 법정동 choropleth 히트맵 / NEW FIND 전용 지도 / 월별 확산 슬라이더 (007 mv_region_pin_density 인프라 활용)

- 동네 태깅 커버리지 100% 유지 (태깅 안 된 데이터는 자산에서 누락)
- 내부 대시보드는 platform_admin 게이트 (055 is_platform_admin + RPC 이중 방어)
- 외부 노출 데이터는 반드시 익명 집계 + k-익명
- 표준 카테고리(음식/카페/명소/문화/기타) 유지 — 자유 태그와 병행
