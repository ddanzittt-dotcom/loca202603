## C. 운영 루틴 — 재수집 스케줄

수집은 전부 **로컬 실행**(문화재 해외 IP 차단 우회 겸). **`npm run ingest:all`**(순차 러너, `-- --dry-run` 지원)이
전 소스를 순서대로 돌리고 `scripts/ingest/logs/`에 로그를 남긴다 — 분기 루틴은 러너 1회 + 표본 검수(A3·A4)로 수렴.

| 스크립트 | 소스 갱신주기 | 재실행 권장 | 비고 |
|---|---|---|---|
| `ingest:festivals` | 수시 등록 | **월 1회** | 종료 축제 자동 정리 포함 |
| `ingest:lifelong` | 분기 | **분기 1회** | 실행 후 **분류 표본 검수**(A4) 필수. 지오코딩 수 분 |
| `ingest:markets` / `ingest:libraries` / `ingest:villages` | 반기~연간 | 반기 1회 | |
| `ingest:parks` / `ingest:trails` | 연간 | 연 1~2회 | trails는 지오코딩 수 분 |
| `ingest:durunubi` | 수시 | 분기 1회 | totalCount 변화 확인(A7) |
| `ingest:heritage` → `ingest:heritage-detail` | 수시 | 반기 1회 | 키 불필요, 국내망에서만. 목록 적재 후 상세 보강 순서 고정(러너에 반영됨) |

- 모든 스크립트는 재실행 = 전체 동기화(upsert + 사라진 행 정리). `--dry-run`으로 필드 변동 먼저 확인.
- **커버리지 운영 지표**: 당분간 러너 로그 보관으로 갈음 — `/admin` 인사이트 표(M)는 필요해질 때.
- 원본 품질 메모: 시도명 오타("경개도" 등)·좌표 결측 행은 소스 문제로 제외 처리 중 — 지자체 데이터라 수용.
